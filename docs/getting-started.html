What inputs OCELOT expects

OCELOT assumes three primary inputs:

  1. a COBRA-compatible metabolic model
  2. a gene expression table such as a gene count matrix with genes as rows and samples as columns
  3. a gene-regulatory network (GRN) inferred by regularized regression, with target genes as rows and TFs as columns

Optional inputs include:

  • an essentiality table for TF knockout benchmarking
  • species-specific medium definitions
  • auto/mixo/heterotrophic configuration settings
  • product reaction IDs for engineering runs

Typical MATLAB setup

initCobraToolbox(false);
changeCobraSolver('gurobi', 'LP');
changeCobraSolver('gurobi', 'MILP');
changeCobraSolverParams('LP', 'feasTol', 1e-9);

For mixed-integer engineering runs, Gurobi should be available for MILP solving.

Minimal unified workflow

load('../Models/iML1515.mat');
expression_data = readtable('../Data/Ecoli_count_matrix.csv', ...
    'ReadRowNames', true, 'VariableNamingRule', 'preserve');
beta_df = readtable('../Data/Ecoli_GRN_lasso.csv', ...
    'ReadRowNames', true);

params.selectedReference = "control__wt_glc";
params.growthType = 'heterotrophic';
params.biomassRxn = 'BIOMASS_Ec_iML1515_core_75p37M';
params.glucoseRxn = 'EX_glc__D_e';
params.glucoseUptake = -10;
params.mediumRxns = ["EX_pi_e","EX_co2_e","EX_fe3_e", ...
                     "EX_h_e","EX_mn2_e","EX_fe2_e"];

params.runKnockout = true;
params.runEngineering = false;

results = OCELOT(model, expression_data, beta_df, params);

Choosing a mode

Knockout mode

Use knockout mode when you want to:

  • simulate the effect of individual TF deletions
  • rank TFs by growth impact
  • compare predicted KO phenotypes against an essentiality benchmark

Engineering mode

Use engineering mode when you want to:

  • search for TF perturbation strategies
  • enforce minimum growth and product constraints
  • retrieve multi-TF strategies and validate them

Common parameters

Parameter Meaning
selectedReference WT condition or reference sample to anchor the analysis
growthType heterotrophic, mixotrophic, or autotrophic
biomassRxn Biomass reaction ID used as the growth objective
thresholdSource Whether thresholds come from TF expression or inferred z values
thresholdSelection Criterion used to choose the final WT threshold
kappaStart / kappaMin Slack penalty range scanned during WT fitting
performEssentialityEvaluation Whether to compute ROC/confusion metrics after KO simulation
validateStrategies Whether to validate engineered TF states after MILP search
validateSingleKOs Whether to test individual KOs extracted from engineering solutions

A first E. coli KO run

params.runKnockout = true;
params.runEngineering = false;
params.performEssentialityEvaluation = true;
params.essentialityTable = essentialityTable;
params.essentialityGeneColumn = 'Gene_ID';
params.essentialityLabelColumn = 'Essentiality';
params.cutoffPerc = 0.1;
params.showPlots = "cutoff";

results = OCELOT(model, expression_data, beta_df, params);

A first engineering run

params.runKnockout = false;
params.runEngineering = true;
params.prodRxn = 'EX_ac_e';
params.maxPerturbations = 5;
params.maxIterations = 12;
params.flexBiomass = 0.5;
params.flexProduct = 0.5;
params.validateStrategies = true;
params.validateSingleKOs = true;

results = OCELOT(model, expression_data, beta_df, params);

Output structure at a glance

The exact nested structure depends on mode, but a typical unified result contains:

  • results.knockout
  • results.engineering
  • results.context
  • results.params

Within these, common objects include:

  • WT_results
  • rmse_table
  • best_threshold
  • all_results
  • metrics_all
  • resultsTable
  • final_results
  • sKO_results

Troubleshooting checklist

  • confirm TF names in beta_df exist in the expression table row names
  • confirm target genes in beta_df overlap with model genes
  • confirm biomass and product reaction IDs match the model exactly
  • check that the selected WT reference exists after expression preprocessing
  • for autotrophic organisms, verify that protected reactions and special constraints are correct
