This page documents the top-level functions that users are expected to call directly. In most projects, OCELOT is the main entry point, while runKnockout and runEngineering are used when only one workflow is needed.

OCELOT

Role. Unified dispatcher for the package. This is the highest-level entry point and is designed for species wrappers, tutorials, and most end-user workflows.

Typical responsibilities.

  • read mode flags from params
  • prepare shared wild-type context
  • dispatch to knockout and/or engineering workflows
  • collect outputs into a single structured result

Use this when. You want one call that can run knockout analysis, engineering analysis, or both, depending on flags.

Important flags.

  • params.runKnockout
  • params.runEngineering
  • params.performEssentialityEvaluation
  • params.validateStrategies
  • params.validateSingleKOs

Expected output pattern. A struct with top-level fields for shared context and optional subfields such as knockout and engineering results.

runKnockout

Role. Execute the TF knockout workflow only.

Typical responsibilities.

  • prepare condition-specific regulatory state
  • simulate one TF knockout at a time
  • store KO biomass and optionally KO flux solutions
  • optionally perform essentiality benchmarking

Use this when. You want TF knockout simulations without running engineering strategy search.

Typical required inputs.

  • configured COBRA model
  • expression table
  • beta matrix
  • params.selectedReference
  • model configuration parameters such as biomass reaction and growth mode

Optional outputs.

  • KO growth table
  • KO fluxes
  • coverage masks
  • evaluation metrics and plots

runEngineering

Role. Execute the engineering workflow only.

Typical responsibilities.

  • reuse or reconstruct WT threshold-selected regulatory context
  • maximize product under growth constraints
  • run the engineering MILP
  • filter raw strategies
  • validate multi-TF strategies and optional single KOs

Use this when. You are searching for TF perturbation strategies rather than benchmarking essentiality.

Typical required inputs.

  • configured COBRA model
  • expression table
  • beta matrix
  • params.prodRxn
  • engineering settings such as perturbation limits and flexibility thresholds

Choosing an entry point

GoalRecommended function
Run the full package from one scriptOCELOT
Simulate TF KOs onlyrunKnockout
Search for TF engineering strategies onlyrunEngineering
Build a species wrapper scriptOCELOT
Debug one path in isolationrunKnockout or runEngineering

Minimal usage patterns

Unified workflow

params.runKnockout = true;
params.runEngineering = true;
results = OCELOT(model, expression_data, beta_df, params);

Knockout-only workflow

params.performEssentialityEvaluation = true;
koResults = runKnockout(model, expression_data, beta_df, params);

Engineering-only workflow

params.validateStrategies = true;
params.validateSingleKOs = true;
engResults = runEngineering(model, expression_data, beta_df, params);
