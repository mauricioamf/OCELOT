Overview

The utility functions keep the core workflow readable by moving repetitive data-cleaning and formatting tasks into small helpers. These functions are usually not called directly by end users, but they are useful when extending the codebase or building new wrappers.

applyBoundChangeList

Apply a structured list of bound edits to a model. This is especially useful for species wrappers or growth-mode-specific setup code where many reactions need consistent lower-bound or upper-bound changes.

Typical input format. A list or struct array describing reaction IDs, values, and which side of the bound should be changed.

Main benefit. centralizes repetitive changeRxnBounds logic.

replaceInfWithFinite

Sanitize vectors or matrices that may contain Inf, -Inf, or otherwise non-finite values introduced during normalization or ratio calculations. The exact replacement strategy may depend on the code path, but the purpose is to keep downstream optimization and plotting code numerically stable.

joinCellString

Join cell arrays of strings into a single display string. This is mainly used for exporting KO or activation lists into compact table columns and for console reporting.

toCellstr

Normalize inputs into MATLAB cell arrays of character vectors. Useful when helper functions may receive strings, chars, or cell arrays and need a stable internal representation.

toStringArray

Normalize inputs into MATLAB string arrays. Useful when wrappers or reports need consistent string-array behavior for indexing, comparison, and table export.

Utility design notes

  • These helpers should remain side-effect free whenever possible.
  • They are good places to add input validation if you want stricter package behavior.
  • If you introduce new species wrappers, prefer reusing these helpers over duplicating formatting or model-edit code.
