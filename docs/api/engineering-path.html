Overview

The engineering path searches for TF perturbation strategies that improve production while satisfying growth and feasibility constraints. It begins from the selected WT regulatory context, constructs a reaction-by-TF influence matrix, solves the strategy MILP, filters raw solutions, and validates the surviving strategy set.

buildUpsilon

Build the reaction-by-TF influence matrix used by the engineering MILP. This matrix translates TF state decisions into reaction-level regulatory influence coefficients.

Typical inputs.

  • reaction-to-target map
  • beta matrix or beta-prime matrix
  • WT regulatory baseline information
  • normalization mode

Why it matters. The MILP uses this matrix directly in its regulatory constraints, so normalization choices can strongly affect solution behavior.

runEngineeringMILP

Core engineering solver. This function usually:

  • builds the optimization variables for fluxes, slacks, and TF states
  • adds steady-state and regulatory constraints
  • imposes growth and product requirements
  • limits the number of TF perturbations
  • iteratively adds integer cuts to enumerate multiple strategies

Typical key parameters.

  • prodRxn
  • maxPerturbations
  • maxIterations
  • flexBiomass
  • flexProduct
  • kappa

filterStrategies

Remove raw strategy rows that are not useful for downstream interpretation. Typical filters include:

  • rows with NaNs in required numeric columns
  • strategies with no perturbations
  • degenerate all-TF patterns
  • solutions with invalid or all-zero TF state vectors

This is mainly a post-processing step to clean up MILP output before validation and export.

validateStrategies

Re-solve the optimization problem with the engineered TF state fixed. This is the main strategy validation step and is used to estimate achievable growth and product levels under the predicted perturbation pattern.

Typical outputs.

  • validated growth and product values under biomass-maximizing or product-maximizing objectives
  • slack usage summaries
  • final export-ready strategy tables

validateSingleKOs

Test individual knockouts extracted from the predicted strategy pool. This is useful for understanding whether one TF alone explains most of the engineered phenotype or whether the effect depends on combinations.

Common outputs. single-KO growth and product ranges, often reported in a compact summary table.

runEngineering

Main engineering workflow. This function usually:

  1. build the WT selected context
  2. compute the reference biomass optimum
  3. maximize product under a growth floor
  4. construct upsilon
  5. run the engineering MILP
  6. filter raw strategies
  7. validate strategies and optional single KOs

Primary outputs.

  • raw and filtered strategy tables
  • validated strategy summaries
  • single-KO validation results
  • bookkeeping objects for WT and product reference solutions
