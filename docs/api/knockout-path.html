Overview

The knockout path takes the selected WT regulatory context and simulates one TF knockout at a time. It can also compare the simulated growth defects against a supplied essentiality reference set to reproduce benchmarking analyses.

buildConditionRegState

Construct the condition-specific regulatory state used for KO simulations. This function usually takes the chosen WT threshold logic and applies it to a particular expression condition to build:

  • active TF state for that condition
  • WT reaction activity baseline
  • KO-specific reaction activity states for each TF
  • coverage indicators describing whether each TF changes at least one reaction

solveKOByRatio

Solve a KO by scaling regulated reaction bounds relative to the WT condition. Rather than rebuilding a complete regulatory model from scratch for each TF, this helper typically computes a KO-to-WT ratio per regulated reaction and solves a slack-penalized optimization problem.

Typical responsibilities.

  • translate KO regulatory effects into bound ratios
  • respect protected reactions
  • build the optimization problem with alpha/beta slack variables
  • return biomass and optionally the full flux solution

runKnockout

Main knockout workflow. This function usually:

  1. prepares WT threshold-selected context
  2. loops through the requested conditions
  3. builds KO regulatory states
  4. solves one TF knockout at a time
  5. stores KO growth, fluxes, and coverage summaries
  6. optionally runs essentiality evaluation and plots

Key parameters.

  • selectedReference
  • performEssentialityEvaluation
  • cutoffPerc
  • showPlots

runEssentialityEvaluation

Benchmark KO predictions against an essentiality table. This function typically computes:

  • continuous scores from KO biomass relative to WT
  • ROC curves and AUC
  • accuracy, sensitivity, and specificity over a cutoff sweep
  • confusion matrices for selected cutoffs

This is the main evaluation layer used when the knockout workflow is intended to reproduce manuscript-style essentiality analyses.

resolveEssentialityColumns

Determine which columns in the essentiality table should be used as the TF or gene identifier column and which column contains the essentiality label. This helper makes evaluation more robust across datasets that use different header names.

plotEvaluation

Render evaluation plots from the knockout benchmarking results. Depending on settings, this may include ROC curves, confusion charts, and aggregate histograms across conditions.

Input expectation. precomputed metrics tables rather than raw KO simulations.

printEvaluationSummary

Print concise summaries of the essentiality benchmarking results to the MATLAB console. Typical outputs include coverage, numbers of essential and non-essential TFs evaluated, and mean accuracy, sensitivity, specificity, and AUC at the selected cutoff.
