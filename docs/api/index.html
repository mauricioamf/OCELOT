Reading the API pages

The pages in this section describe:

  • what each function is responsible for
  • the expected inputs and outputs
  • how functions relate to the rest of the pipeline
  • the most important parameters that affect behavior

Because the code is modular, many helper functions are only meaningful in context. The API notes therefore emphasize where a function fits rather than documenting every local variable.

Entry points

Function Use when
OCELOT You want one unified top-level call
runKnockout You want only KO simulation and optional essentiality benchmarking
runEngineering You want only engineering search and validation

WT setup and selection

Function Purpose
configureBaseModel Build the heterotrophic or generic base model state
configureAutotrophicModel Apply autotrophic-specific model edits
preprocessExpressionData Collapse replicates and prepare condition-level expression tables
buildRegulatoryContext Build TF, target, coefficient, and reaction-mapping context
chooseThresholdValues Generate candidate thresholds
buildThresholdState Build the regulatory state for one threshold
runWTThresholdSelection Scan thresholds and solve WT subproblems
selectBestThreshold Pick the final threshold from the threshold scan

Knockout path

Function Purpose
buildConditionRegState Assemble condition-specific regulatory state
solveKOByRatio Solve a KO problem relative to WT
runKnockout Execute the full knockout workflow
runEssentialityEvaluation Compute ROC/cutoff metrics against a reference set
resolveEssentialityColumns Identify gene and label columns in the essentiality table
plotEvaluation Render evaluation figures
printEvaluationSummary Print evaluation summaries

Engineering path

Function Purpose
buildUpsilon Construct the reaction-by-TF influence matrix
runEngineeringMILP Solve the engineering MILP and enumerate strategies
filterStrategies Remove invalid or uninformative raw strategies
validateStrategies Re-solve with engineered TF states fixed
validateSingleKOs Test individual KOs from the strategy pool
runEngineering Execute the full engineering workflow

Utilities

Function Purpose
applyBoundChangeList Apply a list of bound edits to a model
replaceInfWithFinite Sanitize non-finite numeric values
joinCellString Join cell arrays of strings for export or reporting
toCellstr Normalize input as cellstr
toStringArray Normalize input as string array
